# 家計管理アプリ 要件定義書

> Version 2.0 / 個人用家計管理PWAアプリ

---

## 1. プロジェクト概要

個人用の家計管理アプリをPWA（Progressive Web App）として開発する。iPhoneのホーム画面に追加して、アプリのように起動・操作できることを目的とする。App Storeへのリリースは行わず、完全無料の構成で運用する。

### 1.1 背景・目的

- iOSで使える家計管理アプリを、個人利用目的で自作したい
- App Storeへのリリースやデベロッパー登録費用は発生させたくない
- データ管理は使い慣れたNotionで行い、分析・可視化もNotion側で完結させたい
- 入力は素早く、閲覧・分析はNotionのビュー機能を活用する設計にする

### 1.2 基本方針

- アプリは「入力」と「今月の利用可能額確認」に特化させる
- Notionをデータベース本体として扱う
- 集計・グラフ・ガントチャート等の可視化はNotion側で行う
- 銀行API等との自動連携は行わない（手動入力ベース）
- 一部の定期処理はCloudflare Workers Cronで自動化する

## 2. 技術構成

### 2.1 システム構成

```
iPhone Safari（PWA）
        ↓
Cloudflare Workers（APIプロキシ＋Cron）
        ↓
Notion API
        ↓
Notion DB
```

### 2.2 採用技術

| 項目 | 採用技術 | 備考 |
|---|---|---|
| フロントエンド | HTML / CSS / JavaScript | Figma → コード変換で作成 |
| ホスティング | GitHub Pages | 無料・永続 |
| APIプロキシ | Cloudflare Workers | 無料枠：月10万リクエスト |
| 定期処理 | Cloudflare Workers Cron | 無料枠内で実行 |
| データベース | Notion DB | 無料プランで十分 |
| デザイン | Figma | AI（Claude）でコード化 |
| PWA化 | manifest.json + Service Worker | ホーム画面追加対応 |

### 2.3 費用

全ての構成要素が無料枠で運用可能。想定される利用量では永続的に無料で運用できる。

### 4.3 利用可能額の計算ロジック

給料入力時に以下を計算し、ホーム画面に常時表示：

```
今月の給料
  − 来月支払う固定費合計
  − 来月支払うローン返済合計
  − 来月支払うリボ返済（手動指定額）
  − 今月の目標貯金額
  − 今月の現金・QR決済等の実績支出
  = 今月のクレカ利用可能額
```

クレカ利用を入力するたびに自動で減算。取引を削除・編集した場合は即時再計算。

### 4.4 カード引き落としの自動処理

毎日Cronで各カードの「支払日」をチェックし、該当日なら以下を実行：

1. カードごとに「締め期間内の取引合計」を算出
2. カード引き落としDBに1件作成
3. カード引き落とし明細DBにカード別の内訳を作成
4. 紐づく銀行口座の残高から自動減算
5. ステータスを「確定」に更新

#### 計算式：ローン（元利均等返済）

```
月利 = 年利 ÷ 12
今月の利息 = 現在残債 × 月利
今月の元金 = 月次返済額 − 今月の利息
返済後残債 = 現在残債 − 今月の元金
```

#### 計算式：リボ払い

```
【通常返済時】
利息 = 現在残債 × 年利 ÷ 12
元金 = 今月の支払額（手動指定） − 利息
返済後残債 = 現在残債 − 元金

【繰り上げ返済時】
利息 = 0（全額元金に充当）
返済後残債 = 現在残債 − 返済額
```

#### 計算式：分割手数料（アドオン方式）

```
支払方法DBの分割手数料率表から「回数」に対応する率を取得
手数料総額 = 総額 × 手数料率 × 回数 ÷ 12
月次支払額 = (総額 + 手数料総額) ÷ 回数
```

## 6. 非機能要件

| 項目 | 内容 |
|---|---|
| 対応OS・ブラウザ | iOS Safari（メイン）。他ブラウザは考慮外 |
| オフライン対応 | 入力フォームの表示まではオフラインで可能。保存はオンライン必須 |
| データ保存先 | Notion（マスターデータ・バックアップ兼） |
| セキュリティ | Notion APIキーはCloudflare Workersに環境変数として保持。フロントには露出しない |
| レスポンス性能 | 入力完了までのタップ数を最小化 |
| 多端末対応 | iPhone以外は考慮外 |
| 認証 | 個人利用のため認証なし（URLを知らない前提） |
| Cron実行 | 日本時間を考慮してスケジューリング |

## 7. 運用・保守

- Notion DBへの項目追加・マスタ更新はNotion上で直接行う
- 初期マスタデータ（カテゴリ等）はNotion側で手動入力
- レポート・集計・可視化は全てNotionのビュー機能で実現する
- アプリの更新はGitHubへのプッシュで自動反映（GitHub Pages）
- Cloudflare Workersのコード変更は手動デプロイ
